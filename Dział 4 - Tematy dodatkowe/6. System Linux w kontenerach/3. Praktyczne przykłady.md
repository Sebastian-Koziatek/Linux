# Praktyczne przykłady

## Przykład 1: Uruchomienie serwera www Nginx

```bash
# Pobierz obraz
docker pull nginx

# Uruchom kontener na porcie 8080
docker run -d --name webserver -p 8080:80 nginx

# Sprawdź czy działa
docker ps
curl http://localhost:8080
```

Aby zatrzymać i usunąć:
```bash
docker stop webserver
docker rm webserver
```

## Przykład 2: Eksploracja systemu Ubuntu w kontenerze

```bash
# Uruchom Ubuntu interaktywnie
docker run -it --name moj_ubuntu ubuntu bash
```

Wewnątrz kontenera możesz wykonywać standardowe polecenia Linux:
```bash
ls /
cat /etc/os-release
apt update && apt install -y curl
curl --version
exit
```

## Przykład 3: Uruchomienie bazy danych MySQL

```bash
docker run -d \
  --name baza_mysql \
  -e MYSQL_ROOT_PASSWORD=haslo123 \
  -e MYSQL_DATABASE=testdb \
  -p 3306:3306 \
  mysql:8.0
```

Połączenie z bazą:
```bash
docker exec -it baza_mysql mysql -u root -p
```

## Przykład 4: Montowanie katalogu (volume)

Udostępnienie katalogu z hosta do kontenera:

```bash
mkdir /tmp/dane

docker run -it \
  -v /tmp/dane:/mnt/dane \
  --name test_vol \
  ubuntu bash
```

Wewnątrz kontenera:
```bash
ls /mnt/dane
echo "Plik z kontenera" > /mnt/dane/test.txt
exit
```

Na hoście sprawdź:
```bash
cat /tmp/dane/test.txt
```

## Przykład 5: Kontener działający w tle + logi

```bash
# Uruchom kontener generujący logi
docker run -d --name logger ubuntu bash -c "while true; do echo 'ping'; sleep 2; done"

# Śledź logi na żywo
docker logs -f logger

# Zatrzymaj kontener
docker stop logger && docker rm logger
```

---

## Cykl życia kontenera

```
docker pull       →  pobranie obrazu
docker run        →  utworzenie i uruchomienie kontenera
docker stop       →  zatrzymanie kontenera
docker start      →  ponowne uruchomienie
docker rm         →  usunięcie kontenera
docker rmi        →  usunięcie obrazu
```

## Szybka ściągawka komend

| Zadanie | Komenda |
|---------|---------|
| Pobierz obraz | `docker pull ubuntu` |
| Uruchom kontener interaktywnie | `docker run -it ubuntu bash` |
| Uruchom w tle | `docker run -d nginx` |
| Lista uruchomionych kontenerów | `docker ps` |
| Lista wszystkich kontenerów | `docker ps -a` |
| Lista obrazów | `docker images` |
| Wejdź do kontenera | `docker exec -it NAZWA bash` |
| Zatrzymaj kontener | `docker stop NAZWA` |
| Usuń kontener | `docker rm NAZWA` |
| Usuń obraz | `docker rmi OBRAZ` |
| Logi kontenera | `docker logs NAZWA` |
| Statystyki zasobów | `docker stats` |
| Informacje o kontenerze | `docker inspect NAZWA` |
| Posprzątaj nieużywane zasoby | `docker system prune` |
