# Wprowadzenie do Docker

## Czym jest kontener?

Kontener to **izolowane środowisko uruchomieniowe**, które zawiera aplikację wraz z jej zależnościami (bibliotekami, konfiguracją, systemem plików). Kontenery korzystają z jądra systemu operacyjnego hosta, ale są od siebie odizolowane.

### Kontener vs. maszyna wirtualna

| Cecha | Kontener | Maszyna wirtualna |
|-------|----------|-------------------|
| Rozmiar | Kilkadziesiąt MB | Kilka GB |
| Czas startu | Sekundy | Minuty |
| Izolacja | Na poziomie procesów | Na poziomie sprzętu |
| System operacyjny | Współdzielony kernel | Osobny OS |
| Przenośność | Bardzo wysoka | Ograniczona |

---

## Czym jest Docker?

**Docker** to najpopularniejsza platforma do tworzenia, uruchamiania i zarządzania kontenerami. Składa się z:

- **Docker Engine** – silnik uruchamiający kontenery
- **Docker CLI** – narzędzie wiersza poleceń (`docker`)
- **Docker Hub** – publiczne repozytorium obrazów

---

## Podstawowe pojęcia

| Pojęcie | Opis |
|---------|------|
| **Obraz (Image)** | Szablon kontenera – zawiera system plików i konfigurację |
| **Kontener (Container)** | Uruchomiony egzemplarz obrazu |
| **Docker Hub** | Repozytorium gotowych obrazów (hub.docker.com) |
| **Dockerfile** | Plik z instrukcjami budowania własnego obrazu |
| **Volume** | Trwały magazyn danych kontenera |
| **Port mapping** | Przekierowanie portów między hostem a kontenerem |

## Instalacja Docker

### RHEL / AlmaLinux / Rocky Linux

```bash
sudo dnf install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/rhel/docker-ce.repo
sudo dnf install -y docker-ce docker-ce-cli containerd.io
sudo systemctl enable --now docker
```

### Debian / Ubuntu

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg
curl -fsSL https://get.docker.com | sudo bash
sudo systemctl enable --now docker
```

### Weryfikacja instalacji

```bash
docker --version
sudo docker run hello-world
```

### Uruchamianie Docker bez sudo (opcjonalnie)

```bash
sudo usermod -aG docker $USER
newgrp docker
```

## Jak działa Docker?

```
[Docker CLI] --żądanie--> [Docker Engine (daemon)] --pobiera obraz--> [Docker Hub]
                                    |
                              uruchamia kontener
                                    |
                          [Kontener z aplikacją]
```

1. Użytkownik wydaje polecenie `docker run`
2. Docker sprawdza, czy obraz jest dostępny lokalnie
3. Jeśli nie – pobiera go z Docker Hub
4. Tworzy i uruchamia kontener na podstawie obrazu
