# Podstawowe komendy Docker

## Zarządzanie obrazami

### Pobieranie obrazów

```bash
docker pull ubuntu              # pobierz najnowszy obraz Ubuntu
docker pull ubuntu:22.04        # pobierz konkretną wersję
docker pull nginx               # pobierz obraz serwera Nginx
docker pull alpine              # pobierz lekki Linux Alpine
```

### Listowanie obrazów

```bash
docker images
```

Przykładowy wynik:
```
REPOSITORY   TAG       IMAGE ID       CREATED        SIZE
ubuntu       22.04     3b418d7b466a   2 weeks ago    77.8MB
nginx        latest    a72860cb95fd   3 weeks ago    187MB
alpine       latest    05455a08881e   4 weeks ago    7.04MB
```

### Usuwanie obrazów

```bash
docker rmi nginx                   # usuń obraz po nazwie
docker rmi a72860cb95fd            # usuń obraz po ID
docker rmi $(docker images -q)     # usuń wszystkie obrazy
```

### Wyszukiwanie obrazów

```bash
docker search ubuntu               # szukaj obrazów na Docker Hub
docker search --limit 5 nginx      # ogranicz wyniki do 5
```

## Zarządzanie kontenerami

### Uruchamianie kontenerów

```bash
docker run ubuntu                           # uruchom i zakończ (jeśli brak procesu)
docker run -it ubuntu bash                  # tryb interaktywny z powłoką bash
docker run -d nginx                         # uruchom w tle (detached)
docker run --name moj_serwer nginx          # nadaj nazwę kontenerowi
docker run -p 8080:80 nginx                 # mapowanie portów host:kontener
docker run -v /dane:/data ubuntu            # montowanie wolumenu host:kontener
docker run --rm ubuntu echo "hello"         # usuń kontener po zakończeniu
```

### Listowanie kontenerów

```bash
docker ps                    # pokaż uruchomione kontenery
docker ps -a                 # pokaż wszystkie kontenery (w tym zatrzymane)
docker ps -q                 # pokaż tylko ID kontenerów
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
```

Przykładowy wynik `docker ps -a`:
```
CONTAINER ID   IMAGE     COMMAND                  CREATED          STATUS                      NAMES
a1b2c3d4e5f6   nginx     "/docker-entrypoint.…"   5 minutes ago    Up 5 minutes                moj_serwer
b2c3d4e5f6a1   ubuntu    "bash"                   10 minutes ago   Exited (0) 8 minutes ago    determined_morse
```

### Zatrzymywanie i usuwanie kontenerów

```bash
docker stop moj_serwer           # zatrzymaj kontener (SIGTERM)
docker kill moj_serwer           # wymuś zatrzymanie (SIGKILL)
docker start moj_serwer          # uruchom ponownie zatrzymany kontener
docker restart moj_serwer        # zrestartuj kontener

docker rm moj_serwer             # usuń zatrzymany kontener
docker rm -f moj_serwer          # wymuś usunięcie (nawet uruchomionego)
docker rm $(docker ps -aq)       # usuń wszystkie zatrzymane kontenery
```

## Logowanie do kontenera

### Dostęp do działającego kontenera

```bash
docker exec -it moj_serwer bash     # otwórz bash w działającym kontenerze
docker exec -it moj_serwer sh       # użyj sh (np. dla Alpine Linux)
docker exec moj_serwer ls /var      # wykonaj jednorazowe polecenie
```

### Uruchomienie kontenera w trybie interaktywnym

```bash
docker run -it ubuntu bash           # uruchom Ubuntu i zaloguj się do basha
docker run -it alpine sh             # uruchom Alpine i zaloguj się do sh
```

> Po wpisaniu `exit` lub naciśnięciu **Ctrl+D** wychodzisz z kontenera.  
> Jeśli kontener był uruchomiony z `-it`, zostanie zatrzymany po wyjściu.

### Odłączanie się bez zatrzymania kontenera

Naciśnij: **Ctrl+P, Ctrl+Q** – odłącza terminal bez zatrzymywania kontenera.

Podłącz się ponownie:
```bash
docker attach moj_kontener
```

## Logi i informacje

```bash
docker logs moj_serwer             # pokaż logi kontenera
docker logs -f moj_serwer          # śledź logi na żywo (follow)
docker logs --tail 50 moj_serwer   # ostatnie 50 linii logów

docker inspect moj_serwer          # szczegółowe informacje o kontenerze (JSON)
docker stats                       # zużycie zasobów przez kontenery (na żywo)
docker top moj_serwer              # procesy działające w kontenerze
```

## Informacje o Docker

```bash
docker info                        # informacje o instalacji Docker
docker version                     # wersja Docker client i server
docker system df                   # użycie dysku przez Docker
docker system prune                # usuń nieużywane zasoby (kontenery, obrazy, sieci)
```
