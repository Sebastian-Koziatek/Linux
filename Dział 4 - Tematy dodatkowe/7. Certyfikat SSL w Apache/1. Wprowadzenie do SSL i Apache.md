# Wprowadzenie do SSL i Apache

## Czym jest certyfikat SSL/TLS?

Certyfikat SSL/TLS to plik zawierający klucz publiczny i informacje o właścicielu domeny. Umożliwia szyfrowanie połączeń między przeglądarką a serwerem (HTTPS).

### Składowe certyfikatu

| Plik | Opis | Typowe rozszerzenie |
|------|------|---------------------|
| **Klucz prywatny** | Tajny klucz serwera – nigdy nie opuszcza serwera | `.key` |
| **Certyfikat** | Klucz publiczny podpisany przez CA | `.crt`, `.pem` |
| **Łańcuch CA** | Certyfikaty pośrednie urzędu certyfikacji | `.ca-bundle`, `.pem` |
| **CSR** | Żądanie podpisania certyfikatu – wysyłane do CA | `.csr` |

---

## Rodzaje certyfikatów

| Typ | Opis | Koszt |
|-----|------|-------|
| **Self-signed** | Podpisany samodzielnie – przeglądarka wyświetla ostrzeżenie | Bezpłatny |
| **Let's Encrypt** | Darmowy certyfikat od zaufanego CA, ważny 90 dni | Bezpłatny |
| **DV (Domain Validation)** | Weryfikacja tylko domeny | Od ~20 USD/rok |
| **OV (Organization Validation)** | Weryfikacja domeny i organizacji | Od ~100 USD/rok |
| **EV (Extended Validation)** | Pełna weryfikacja organizacji | Od ~200 USD/rok |

---

## Wymagania wstępne

Przed podmianą certyfikatu upewnij się, że:

```bash
# Apache jest zainstalowany
httpd -v                        # RHEL/AlmaLinux
apache2 -v                      # Debian/Ubuntu

# Moduł SSL jest włączony
httpd -M | grep ssl             # RHEL
apache2ctl -M | grep ssl        # Debian/Ubuntu

# Jeśli moduł SSL nie jest aktywny (Debian/Ubuntu)
sudo a2enmod ssl
sudo systemctl restart apache2
```

### Instalacja mod_ssl (RHEL/AlmaLinux)

```bash
sudo dnf install mod_ssl -y
sudo systemctl restart httpd
```

## Lokalizacje plików konfiguracyjnych Apache

### RHEL / AlmaLinux / Rocky Linux

| Ścieżka | Opis |
|---------|------|
| `/etc/httpd/conf/httpd.conf` | Główny plik konfiguracyjny |
| `/etc/httpd/conf.d/ssl.conf` | Konfiguracja SSL |
| `/etc/httpd/conf.d/*.conf` | Pliki VirtualHost |
| `/etc/pki/tls/certs/` | Katalog certyfikatów |
| `/etc/pki/tls/private/` | Katalog kluczy prywatnych |

### Debian / Ubuntu

| Ścieżka | Opis |
|---------|------|
| `/etc/apache2/apache2.conf` | Główny plik konfiguracyjny |
| `/etc/apache2/sites-available/` | Dostępne konfiguracje VirtualHost |
| `/etc/apache2/sites-enabled/` | Aktywne konfiguracje (symlinki) |
| `/etc/ssl/certs/` | Katalog certyfikatów |
| `/etc/ssl/private/` | Katalog kluczy prywatnych |
