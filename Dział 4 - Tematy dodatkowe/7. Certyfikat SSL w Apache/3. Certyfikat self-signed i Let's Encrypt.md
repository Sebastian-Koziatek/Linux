# Certyfikat self-signed i Let's Encrypt

## Certyfikat self-signed (samopodpisany)

Przydatny na środowiskach testowych i wewnętrznych, gdzie nie potrzeba zaufanego CA.

### Generowanie certyfikatu self-signed

```bash
# Generuj klucz prywatny i certyfikat ważny 365 dni
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/pki/tls/private/serwer.key \
  -out    /etc/pki/tls/certs/serwer.crt
```

Zostaniesz poproszony o dane:
```
Country Name (2 letter code) [XX]: PL
State or Province Name: Mazowieckie
Locality Name: Warszawa
Organization Name: Moja Firma
Organizational Unit Name: IT
Common Name (domain): twojadomena.pl
Email Address: admin@twojadomena.pl
```

### Generowanie bez pytań (nieinteraktywnie)

```bash
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/pki/tls/private/serwer.key \
  -out    /etc/pki/tls/certs/serwer.crt \
  -subj "/C=PL/ST=Mazowieckie/L=Warszawa/O=Moja Firma/CN=twojadomena.pl"
```

### Ustawienie uprawnień

```bash
sudo chmod 644 /etc/pki/tls/certs/serwer.crt
sudo chmod 600 /etc/pki/tls/private/serwer.key
```

## Generowanie CSR (dla certyfikatu od CA)

Jeśli kupujesz certyfikat od urzędu certyfikacji, musisz wygenerować CSR:

```bash
# Krok 1: Generuj klucz prywatny
sudo openssl genrsa -out /etc/pki/tls/private/serwer.key 2048

# Krok 2: Generuj CSR
sudo openssl req -new \
  -key /etc/pki/tls/private/serwer.key \
  -out /tmp/serwer.csr \
  -subj "/C=PL/ST=Mazowieckie/L=Warszawa/O=Moja Firma/CN=twojadomena.pl"

# Wyświetl CSR (skopiuj i wyślij do CA)
cat /tmp/serwer.csr
```

## Let's Encrypt – darmowy certyfikat od zaufanego CA

Let's Encrypt wystawia certyfikaty ważne **90 dni** z automatycznym odnawianiem.

### Instalacja Certbota

```bash
# RHEL / AlmaLinux
sudo dnf install epel-release -y
sudo dnf install certbot python3-certbot-apache -y

# Debian / Ubuntu
sudo apt install certbot python3-certbot-apache -y
```

### Wydanie certyfikatu dla domeny

```bash
sudo certbot --apache -d twojadomena.pl -d www.twojadomena.pl
```

Certbot automatycznie:
- zweryfikuje własność domeny
- wygeneruje certyfikat
- zaktualizuje konfigurację Apache
- skonfiguruje przekierowanie HTTP → HTTPS

### Lokalizacja plików Let's Encrypt

```
/etc/letsencrypt/live/twojadomena.pl/cert.pem       # certyfikat
/etc/letsencrypt/live/twojadomena.pl/privkey.pem     # klucz prywatny
/etc/letsencrypt/live/twojadomena.pl/chain.pem       # łańcuch CA
/etc/letsencrypt/live/twojadomena.pl/fullchain.pem   # certyfikat + łańcuch
```

### Odnawianie certyfikatu

```bash
# Test odnowienia (bez faktycznej zmiany)
sudo certbot renew --dry-run

# Ręczne odnowienie
sudo certbot renew

# Automatyczne odnowienie przez cron (dodaj do crontab)
0 3 * * * certbot renew --quiet && systemctl reload httpd
```

### Sprawdzenie ważności wszystkich certyfikatów Let's Encrypt

```bash
sudo certbot certificates
```

## Diagnostyka problemów

### Apache nie startuje po zmianie certyfikatu

```bash
# Sprawdź logi błędów
sudo journalctl -xe -u httpd        # RHEL
sudo journalctl -xe -u apache2      # Debian

sudo tail -50 /var/log/httpd/error_log      # RHEL
sudo tail -50 /var/log/apache2/error.log    # Debian
```

### Klucz nie pasuje do certyfikatu

```bash
# Porównaj sumy kontrolne (muszą być identyczne)
openssl x509 -noout -modulus -in serwer.crt | md5sum
openssl rsa  -noout -modulus -in serwer.key | md5sum
```

### Certyfikat wygasł

```bash
# Sprawdź datę ważności
openssl x509 -in /etc/pki/tls/certs/serwer.crt -noout -dates

# Sprawdź przez sieć
openssl s_client -connect twojadomena.pl:443 </dev/null 2>/dev/null \
  | openssl x509 -noout -dates
```

### SELinux blokuje dostęp do pliku klucza

```bash
# Sprawdź kontekst SELinux
ls -Z /etc/pki/tls/private/serwer.key

# Ustaw właściwy kontekst
sudo restorecon -v /etc/pki/tls/private/serwer.key
sudo chcon -t cert_t /etc/pki/tls/private/serwer.key
```
