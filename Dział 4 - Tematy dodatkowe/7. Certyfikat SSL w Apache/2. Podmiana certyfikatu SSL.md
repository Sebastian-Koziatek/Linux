# Podmiana certyfikatu SSL w Apache

## Krok 1: Przygotowanie nowych plików certyfikatu

Przed rozpoczęciem upewnij się, że masz:
- nowy certyfikat: `nowy_certyfikat.crt`
- klucz prywatny: `nowy_klucz.key`
- (opcjonalnie) łańcuch CA: `ca-bundle.crt`

### Weryfikacja plików przed podmianą

```bash
# Sprawdź czy certyfikat jest prawidłowy
openssl x509 -in nowy_certyfikat.crt -text -noout | grep -E "Subject|Issuer|Not (Before|After)"

# Sprawdź datę ważności
openssl x509 -in nowy_certyfikat.crt -noout -dates

# Sprawdź czy klucz pasuje do certyfikatu (wyniki powinny być identyczne)
openssl x509 -in nowy_certyfikat.crt -noout -modulus | md5sum
openssl rsa  -in nowy_klucz.key     -noout -modulus | md5sum

# Sprawdź czy certyfikat pasuje do domeny
openssl x509 -in nowy_certyfikat.crt -noout -subject
```

## Krok 2: Kopia zapasowa starych certyfikatów

Zawsze rób backup przed podmianą:

```bash
# RHEL / AlmaLinux
sudo cp /etc/pki/tls/certs/serwer.crt   /etc/pki/tls/certs/serwer.crt.bak
sudo cp /etc/pki/tls/private/serwer.key /etc/pki/tls/private/serwer.key.bak

# Debian / Ubuntu
sudo cp /etc/ssl/certs/serwer.crt   /etc/ssl/certs/serwer.crt.bak
sudo cp /etc/ssl/private/serwer.key /etc/ssl/private/serwer.key.bak

# Backup konfiguracji Apache
sudo cp /etc/httpd/conf.d/ssl.conf /etc/httpd/conf.d/ssl.conf.bak          # RHEL
sudo cp /etc/apache2/sites-available/default-ssl.conf /etc/apache2/sites-available/default-ssl.conf.bak  # Debian
```

## Krok 3: Skopiowanie nowych plików

### RHEL / AlmaLinux / Rocky Linux

```bash
# Skopiuj certyfikat
sudo cp nowy_certyfikat.crt /etc/pki/tls/certs/serwer.crt

# Skopiuj klucz prywatny
sudo cp nowy_klucz.key /etc/pki/tls/private/serwer.key

# Skopiuj łańcuch CA (jeśli dotyczy)
sudo cp ca-bundle.crt /etc/pki/tls/certs/ca-bundle.crt

# Ustaw właściwe uprawnienia
sudo chmod 644 /etc/pki/tls/certs/serwer.crt
sudo chmod 600 /etc/pki/tls/private/serwer.key
sudo chown root:root /etc/pki/tls/private/serwer.key
```

### Debian / Ubuntu

```bash
sudo cp nowy_certyfikat.crt /etc/ssl/certs/serwer.crt
sudo cp nowy_klucz.key      /etc/ssl/private/serwer.key
sudo cp ca-bundle.crt       /etc/ssl/certs/ca-bundle.crt

sudo chmod 644 /etc/ssl/certs/serwer.crt
sudo chmod 600 /etc/ssl/private/serwer.key
sudo chown root:ssl-cert /etc/ssl/private/serwer.key
```

## Krok 4: Konfiguracja VirtualHost SSL

Sprawdź lub zaktualizuj konfigurację Apache, aby wskazywała na nowe pliki.

### Przykładowy VirtualHost (RHEL: `/etc/httpd/conf.d/ssl.conf`)

```apache
<VirtualHost *:443>
    ServerName twojadomena.pl
    DocumentRoot /var/www/html

    SSLEngine on

    # Certyfikat serwera
    SSLCertificateFile      /etc/pki/tls/certs/serwer.crt

    # Klucz prywatny
    SSLCertificateKeyFile   /etc/pki/tls/private/serwer.key

    # Łańcuch certyfikatów CA (jeśli dotyczy)
    SSLCertificateChainFile /etc/pki/tls/certs/ca-bundle.crt

    # Zalecane ustawienia bezpieczeństwa
    SSLProtocol             all -SSLv3 -TLSv1 -TLSv1.1
    SSLCipherSuite          ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256
    SSLHonorCipherOrder     off

    ErrorLog  /var/log/httpd/ssl_error.log
    CustomLog /var/log/httpd/ssl_access.log combined
</VirtualHost>
```

### Przykładowy VirtualHost (Debian: `/etc/apache2/sites-available/default-ssl.conf`)

```apache
<VirtualHost *:443>
    ServerName twojadomena.pl
    DocumentRoot /var/www/html

    SSLEngine on

    SSLCertificateFile      /etc/ssl/certs/serwer.crt
    SSLCertificateKeyFile   /etc/ssl/private/serwer.key
    SSLCertificateChainFile /etc/ssl/certs/ca-bundle.crt

    SSLProtocol             all -SSLv3 -TLSv1 -TLSv1.1
</VirtualHost>
```

## Krok 5: Sprawdzenie konfiguracji i restart Apache

```bash
# Sprawdź składnię konfiguracji Apache (przed restartem!)
sudo apachectl configtest          # Debian/Ubuntu
sudo httpd -t                      # RHEL/AlmaLinux

# Prawidłowy wynik:
# Syntax OK

# Restart Apache
sudo systemctl restart httpd       # RHEL/AlmaLinux
sudo systemctl restart apache2     # Debian/Ubuntu
```

> **Ważne:** Zawsze uruchom `configtest` przed restartem – błąd konfiguracji spowoduje, że Apache nie wystartuje.

## Krok 6: Weryfikacja po podmiance

```bash
# Sprawdź certyfikat przez OpenSSL
openssl s_client -connect twojadomena.pl:443 -showcerts </dev/null 2>/dev/null \
  | openssl x509 -noout -dates -subject

# Sprawdź certyfikat przez curl
curl -vI https://twojadomena.pl 2>&1 | grep -E "SSL|certificate|expire|issuer"

# Sprawdź status Apache
sudo systemctl status httpd        # RHEL
sudo systemctl status apache2      # Debian
```

## Krok 7: Przekierowanie HTTP → HTTPS (opcjonalnie)

Dodaj do konfiguracji na porcie 80, żeby wymuszać HTTPS:

```apache
<VirtualHost *:80>
    ServerName twojadomena.pl
    Redirect permanent / https://twojadomena.pl/
</VirtualHost>
```

Lub nowocześniejszym sposobem:

```apache
<VirtualHost *:80>
    ServerName twojadomena.pl
    RewriteEngine On
    RewriteRule ^(.*)$ https://%{HTTP_HOST}$1 [R=301,L]
</VirtualHost>
```
