# Przełączniki i opcje nmap

## Określanie celu skanowania

```bash
nmap 192.168.1.1              # pojedynczy host
nmap 192.168.1.1-50           # zakres adresów
nmap 192.168.1.0/24           # cała podsieć
nmap host1 host2 host3        # wiele hostów
nmap -iL lista_hostow.txt     # hosty z pliku
```

---

## Typy skanowania portów (`-s`)

| Przełącznik | Nazwa | Opis |
|-------------|-------|------|
| `-sT` | TCP Connect | Pełne połączenie TCP – wolniejsze, widoczne w logach |
| `-sS` | TCP SYN (Stealth) | Skanowanie półotwarte – szybsze, mniej widoczne |
| `-sU` | UDP | Skanowanie portów UDP (wolniejsze) |
| `-sA` | TCP ACK | Wykrywa reguły firewalla |
| `-sN` | TCP Null | Wysyła pakiet bez flag TCP |
| `-sF` | TCP FIN | Wysyła pakiet z flagą FIN |
| `-sX` | Xmas | Pakiet z flagami FIN, PSH, URG |

### Przykład – skanowanie SYN (najczęściej używane):
```bash
sudo nmap -sS 192.168.1.1
```

---

## Określanie portów (`-p`)

```bash
nmap -p 22 192.168.1.1            # tylko port 22
nmap -p 22,80,443 192.168.1.1     # konkretne porty
nmap -p 1-1000 192.168.1.1        # zakres portów
nmap -p- 192.168.1.1              # wszystkie 65535 portów
nmap --top-ports 100 192.168.1.1  # top 100 najczęstszych portów
```

---

## Wykrywanie systemu operacyjnego i wersji usług

| Przełącznik | Opis |
|-------------|------|
| `-O` | Wykrywanie systemu operacyjnego |
| `-sV` | Wykrywanie wersji usług |
| `-A` | Wszystko naraz: OS, wersje, skrypty, traceroute |

```bash
sudo nmap -O 192.168.1.1          # wykryj system operacyjny
sudo nmap -sV 192.168.1.1         # wykryj wersje usług
sudo nmap -A 192.168.1.1          # pełny profil hosta
```

---

## Prędkość skanowania (`-T`)

Nmap oferuje 6 poziomów prędkości:

| Przełącznik | Nazwa | Opis |
|-------------|-------|------|
| `-T0` | Paranoid | Bardzo wolny, omija IDS |
| `-T1` | Sneaky | Wolny |
| `-T2` | Polite | Delikatny dla sieci |
| `-T3` | Normal | Domyślny |
| `-T4` | Aggressive | Szybszy, wymaga dobrego łącza |
| `-T5` | Insane | Bardzo szybki, ryzyko błędów |

```bash
nmap -T4 192.168.1.0/24    # szybkie skanowanie podsieci
```

---

## Wykrywanie hostów (`-P`)

```bash
nmap -sn 192.168.1.0/24    # tylko ping sweep – bez skanowania portów
nmap -Pn 192.168.1.1       # pomija wykrywanie hosta (traktuje jako aktywny)
nmap -PS 192.168.1.1       # TCP SYN ping
nmap -PA 192.168.1.1       # TCP ACK ping
nmap -PU 192.168.1.1       # UDP ping
```

---

## Skrypty NSE (`--script`)

Nmap Scripting Engine (NSE) pozwala uruchamiać gotowe skrypty do zaawansowanych zadań:

```bash
nmap --script default 192.168.1.1         # domyślny zestaw skryptów
nmap --script vuln 192.168.1.1            # sprawdzenie podatności
nmap --script http-title 192.168.1.1      # pobierz tytuł strony HTTP
nmap --script banner 192.168.1.1          # pobierz banner usługi
nmap --script ssh-auth-methods 192.168.1.1  # metody uwierzytelnienia SSH
```

---

## Zapis wyników (`-o`)

```bash
nmap -oN wyniki.txt 192.168.1.1       # format normalny (czytelny)
nmap -oX wyniki.xml 192.168.1.1       # format XML
nmap -oG wyniki.gnmap 192.168.1.1     # format grepable
nmap -oA wyniki 192.168.1.1           # wszystkie formaty naraz
```

---

## Pozostałe przydatne przełączniki

| Przełącznik | Opis |
|-------------|------|
| `-v` | Tryb szczegółowy (verbose) |
| `-vv` | Bardziej szczegółowy |
| `-d` | Tryb debugowania |
| `--open` | Pokaż tylko otwarte porty |
| `--reason` | Pokaż powód stanu portu |
| `-6` | Skanowanie IPv6 |
| `--exclude 192.168.1.5` | Wyklucz dany host |
| `--excludefile plik.txt` | Wyklucz hosty z pliku |
| `-n` | Nie wykonuj rozwiązywania nazw DNS |
| `-R` | Zawsze rozwiązuj nazwy DNS |
