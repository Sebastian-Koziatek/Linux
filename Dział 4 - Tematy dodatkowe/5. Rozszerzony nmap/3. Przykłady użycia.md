# Przykłady użycia nmap

## Scenariusze praktyczne

### 1. Szybkie sprawdzenie aktywnych hostów w sieci lokalnej

```bash
sudo nmap -sn 192.168.1.0/24
```

Wynik:
```
Nmap scan report for 192.168.1.1
Host is up (0.0025s latency).
Nmap scan report for 192.168.1.10
Host is up (0.0041s latency).
Nmap done: 256 IP addresses (2 hosts up) scanned in 2.54 seconds
```

---

### 2. Skanowanie popularnych portów z wykryciem usług

```bash
sudo nmap -sV --top-ports 20 192.168.1.10
```

---

### 3. Pełny profil jednego serwera

```bash
sudo nmap -A -T4 192.168.1.10
```

Pokazuje:
- otwarte porty i usługi
- wersje oprogramowania
- system operacyjny
- traceroute

---

### 4. Sprawdzenie czy SSH jest dostępne w całej sieci

```bash
sudo nmap -p 22 --open 192.168.1.0/24
```

---

### 5. Skanowanie wszystkich portów TCP

```bash
sudo nmap -p- -T4 192.168.1.10
```

> Uwaga: skanowanie 65535 portów trwa kilka minut.

---

### 6. Wykrywanie systemu operacyjnego

```bash
sudo nmap -O 192.168.1.10
```

Wynik (fragment):
```
OS details: Linux 4.15 - 5.6
```

---

### 7. Sprawdzenie wersji usług na konkretnych portach

```bash
sudo nmap -sV -p 22,80,443,3306 192.168.1.10
```

---

### 8. Skanowanie z zapisem wyników

```bash
sudo nmap -sV -T4 -oA skan_serwer 192.168.1.10
```

Tworzy trzy pliki: `skan_serwer.nmap`, `skan_serwer.xml`, `skan_serwer.gnmap`

---

### 9. Skanowanie UDP (wolniejsze)

```bash
sudo nmap -sU --top-ports 20 192.168.1.10
```

Często używane do sprawdzenia portów DNS (53), SNMP (161), DHCP (67/68).

---

### 10. Użycie skryptu do wykrycia podatności

```bash
sudo nmap --script vuln -p 80,443 192.168.1.10
```

---

## Zestawienie – kombinacje przełączników na szkoleniu

| Cel | Polecenie |
|-----|-----------|
| Kto jest w sieci? | `sudo nmap -sn 192.168.1.0/24` |
| Jakie porty ma serwer? | `sudo nmap -p- 192.168.1.10` |
| Jakie usługi działają? | `sudo nmap -sV 192.168.1.10` |
| Jaki system operacyjny? | `sudo nmap -O 192.168.1.10` |
| Szybki audyt jednego hosta | `sudo nmap -A -T4 192.168.1.10` |
| Zapisz wyniki do pliku | `sudo nmap -oA wyniki 192.168.1.10` |
