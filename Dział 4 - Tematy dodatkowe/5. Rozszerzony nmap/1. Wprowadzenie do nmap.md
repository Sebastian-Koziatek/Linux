# Wprowadzenie do nmap

## Czym jest nmap?

`nmap` (Network Mapper) to jedno z najpopularniejszych narzędzi do skanowania sieci i audytu bezpieczeństwa. Umożliwia wykrywanie hostów, otwartych portów, wersji usług oraz systemu operacyjnego.

Nmap jest **darmowy i open-source**, działa na systemach Linux, Windows i macOS.

## Instalacja

### RHEL / AlmaLinux / Rocky Linux
```bash
sudo dnf install nmap -y
```

### Debian / Ubuntu
```bash
sudo apt install nmap -y
```

### Weryfikacja instalacji
```bash
nmap --version
```

## Podstawowe pojęcia

| Pojęcie | Opis |
|---------|------|
| **Host** | Urządzenie w sieci (komputer, serwer, router) |
| **Port** | Punkt komunikacyjny (0–65535) |
| **Usługa** | Program nasłuchujący na danym porcie (np. SSH na 22) |
| **Stan portu** | `open`, `closed`, `filtered` |

### Stany portów

- **open** – port jest otwarty, usługa nasłuchuje
- **closed** – port dostępny, ale żadna usługa nie nasłuchuje
- **filtered** – nmap nie może określić stanu (firewall blokuje pakiety)
- **unfiltered** – port dostępny, ale stan nieznany
- **open|filtered** – nmap nie może odróżnić open od filtered

## Pierwsze skanowanie

```bash
nmap 192.168.1.1
```

Wynik pokazuje otwarte porty na podanym hoście. Przykładowy wynik:
```
Starting Nmap 7.94 ...
PORT     STATE SERVICE
22/tcp   open  ssh
80/tcp   open  http
443/tcp  open  https
```

> **Ważne:** Skanowanie sieci bez zgody właściciela jest nielegalne. Używaj nmap tylko na własnych systemach lub po uzyskaniu pisemnej zgody.
